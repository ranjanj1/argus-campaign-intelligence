Skills are access control profiles — they control which data collections the RAG pipeline is allowed to search. Think of them as user roles:

  ┌────────────────┬────────────────────────┬──────────────────────────────────────────────────┐
  │     Skill      │      Who it's for      │            Collections it can search             │
  ├────────────────┼────────────────────────┼──────────────────────────────────────────────────┤
  │ All Campaigns  │ Internal analyst (you) │ All 6 collections                                │
  ├────────────────┼────────────────────────┼──────────────────────────────────────────────────┤
  │ Own Campaigns  │ Client portal user     │ campaign_performance, ad_copy, audience_segments │
  ├────────────────┼────────────────────────┼──────────────────────────────────────────────────┤
  │ Strategy Docs  │ C-suite / executive    │ monthly_reports, strategy_briefs, budget         │
  ├────────────────┼────────────────────────┼──────────────────────────────────────────────────┤
  │ Performance    │ Performance analyst    │ campaign_performance, audience_segments          │
  ├────────────────┼────────────────────────┼──────────────────────────────────────────────────┤
  │ Ad Copy Review │ Creative team          │ ad_copy_library, campaign_performance            │
  ├────────────────┼────────────────────────┼──────────────────────────────────────────────────┤
  │ Budget Data    │ Finance                │ budget_allocations, monthly_reports              │
  └────────────────┴────────────────────────┴──────────────────────────────────────────────────┘

  So if you're on Budget Data skill and ask "what are the best ad creatives?", the RAG pipeline literally cannot see the ad_copy_library — it's blocked at the
  collection level.

  You have 4 seed clients, each with the same 6 file types:
  - acme_corp — Retail
  - techflow — SaaS
  - greenleaf — Ecommerce
  - northstar — Finance

  Each client has: campaign_performance.csv, ad_copy_library.csv, audience_segments.csv, strategy_brief.docx, budget_allocation.xlsx, monthly_report.pdf

  ---
  How to Test Skills

  Since auth is disabled in dev mode, you can switch skills in the sidebar and ask the same question to see different results:

  Test 1 — Data isolation by skill:
  1. Switch to Budget Data skill → ask "what are our top performing ads?"
    - Should say it can't find ad data (only sees budget/reports)
  2. Switch to All Campaigns → ask the same question
    - Should return full ad copy results

  Test 2 — Client isolation:
  1. Switch client to TechFlow → ask "what is our ROAS?"
    - Should only return TechFlow data
  2. Switch to Acme Corp → same question
    - Should return Acme data only

  Test 3 — via curl with a real JWT:
  # Generate a token for a specific skill (run in Python)
  ARGUS_PROFILES=develop python -c "
  from argus.server.utils.auth import create_token
  from argus.utils.skills import ClientSkill
  print(create_token(ClientSkill.BUDGET, 'acme_corp'))
  "

  # Use it
  curl -X POST http://localhost:8001/v1/chat \
    -H 'Authorization: Bearer <token>' \
    -H 'Content-Type: application/json' \
    -d '{"message": "what are the best ad creatives?", "stream": false}'
